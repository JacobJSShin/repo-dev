# requirements.md
요건 정의

## 화면 요건 
- 다크/라이트 모드 전환 가능해야 함
- 좌측 세로 메뉴창 생성, 우측 상단은 결과창 생성, 우측 하단은 시각화 결과창 생성
- 화면의 구도는 assets/pagetemp.png 파일 참고
- 가수명 입력창 생성
- 가수명을 입력 받아 입력 받은 가수명과 유사한 가수의 아티스트명을 artist 메타 테이블에서 조회하여 화면에 출력 
- 출력된 아티스트명 중 조회하고자 하는 아티스트를 선택하도록 하고 중복을 허용 함 
- 최종 선택된 아티스트의 artist_id를 기준으로 사용자별 아티스트별 청취건수 테이블을 조회
- 추가로 조회 조건을 적용할 수 있는 필터 창 생성 
- 필터의 종류 
    - 청취건수
- 조회한 결과 데이터의 summary를 하단에 출력
- summary에 대한 시각화 보드 생성
- 사용자별 아티스트별 청취건수 데이터를 csv로 다운로드할 수 있는 버튼 생성


## 기능 요건
- 사용자 입력 
    - 가수명
    - 청취건수 필터 
- 유사한 가수의 아티스트명 
    - 입력값에 대한 LIKE 조건으로 조회 
    - 조회 결과를 5개까지 화면에 보여줌
    - 조회 결과 중 중복 선택 가능해야 함 
    - 입력값을 받아 조회하는 쿼리는 별도 제공 예정
- 조회한 결과 데이터의 summary
    - 전체 건수 (사용자수)
    - 청취 건수 최대값
    - 청취 건수 최소값
    - 청취 건수 평균값
    - 청취건수 중앙값 (백분위의 50위값)
- 조회 결과 출력 
    - 조회 결과 데이터의 화면 출력 (청취건수 많은 순 상위 20건만)
    - 조회 결과 데이터 쿼리는 별도 제공 예정
- 조회 결과 데이터 다운로드 기능
    - 다운로드 버튼으로 CSV 다운로드 
    - 화면 실행한 사용자의 로컬 PC에 다른이름으로 저장 형태로 제공
